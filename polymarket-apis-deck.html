The API Landscape

Polymarket exposes data through multiple distinct services. Here are the primary ones.

Gamma API

Market metadata & discovery

gamma-api.polymarket.com

Hosted indexing service for browsing markets. Returns questions, slugs, outcomes, volume, liquidity, tags, events, and sports metadata.
Public

CLOB API

Order book, prices & trading

clob.polymarket.com

Hybrid-decentralized Central Limit Order Book. Off-chain matching, on-chain settlement. Two-tier auth system (L1: EIP-712, L2: HMAC).
Read: Public Trade: L1+L2

Data API

Users, trades, rankings & more

data-api.polymarket.com

The most extensive API. Positions, trades, P&L, leaderboard, pricing, orderbook summaries, profiles, search, comments, and accounting snapshots.
Mostly Public

Bridge API

Deposits & withdrawals

Fund your Polymarket wallet. Supports multiple chains (EVM, Solana, Bitcoin). Provides quotes with fee breakdowns.
Auth Required

WebSocket

Real-time streaming

wss://ws-subscriptions-clob.polymarket.com

Live orderbook updates, price changes, trade notifications, and user order status via WebSocket channels.
WSS Market: Public User: Auth

Strapi API

Activity feed (legacy)

strapi-matic.poly.market

Older activity feed. Useful as a fallback when other APIs are unavailable.
Public

CLOB Authentication — Two-Tier System

The CLOB uses two levels of auth. L1 proves wallet ownership; L2 signs API requests for speed.  Verified via docs

Level 1 — EIP-712 Signature

What: Signs messages with your private key using Ethereum's EIP-712 typed data standard. Proves wallet ownership non-custodially.

Used for: Generating API credentials, initial authentication.

L1 Headers

POLY_ADDRESS:    <polygon_address>
POLY_SIGNATURE:  <eip712_signature>
POLY_TIMESTAMP:  <unix_timestamp>
POLY_NONCE:      0

Generate API Credentials

// Create new creds (L1 auth required)
POST /auth/api-key

// Derive existing creds
GET  /auth/derive-api-key

// Response:
{
  "apiKey":     "...",
  "secret":     "...",  // base64-encoded
  "passphrase": "..."
}

Level 2 — HMAC-SHA256

What: Signs each API request with HMAC-SHA256 using the credentials from L1. Faster than signing with private key each time.

Used for: All trading operations (orders, cancels). Note: order payloads still require the private key signature too.

L2 Headers

POLY_API_KEY:    <api_key>
POLY_SIGNATURE:  <hmac_sha256_sig>
POLY_TIMESTAMP:  <unix_timestamp>
POLY_PASSPHRASE: <passphrase>

HMAC Signature

import hmac, hashlib, time

timestamp = str(int(time.time()))
message = f"{timestamp}{method}{path}{body}"

signature = hmac.new(
    base64.b64decode(api_secret),
    message.encode('utf-8'),
    hashlib.sha256
).hexdigest()
Signature Types: When initializing the client, declare your wallet type — EOA (standard), POLY_PROXY (Polymarket wallet), or GNOSIS_SAFE (multisig).
Non-custodial: Private key never leaves your control. All trading activity remains non-custodial.

Conditional Token Framework (CTF)

Markets on Polymarket use the Conditional Token Framework — an on-chain protocol for creating and trading conditional outcome tokens.

OperationWhat It Does
SplitConvert USDC into YES + NO tokens for a market
MergeCombine YES + NO tokens back into USDC
RedeemAfter resolution, exchange winning tokens for USDC
 
// How YES/NO tokens work:
//
// 1 USDC  --split-->  1 YES + 1 NO
// 1 YES + 1 NO  --merge-->  1 USDC
//
// If market resolves YES:
//   1 YES  --redeem-->  1 USDC
//   1 NO   --redeem-->  0 USDC
//
// This is why YES + NO prices sum to ~$1
⚠️

The Field Naming Problem

The biggest gotcha: the same concept has different field names across APIs and even across endpoints within the same API. You must normalize.

Wallet Address

// All of these mean "wallet":
wallet = (
    trade.get("maker")        // CLOB
    or trade.get("user")    // Data API
    or trade.get("proxyWallet") // Leaderboard
    or trade.get("taker")   // CLOB alt
    or trade.get("address") // Profiles
)

Market Identifier

// Three names for the same thing:
market = (
    trade.get("conditionId")  // Gamma, CTF
    or trade.get("market")   // CLOB
    or trade.get("marketId") // Data API
)

Trade Size

// Size vs Amount vs computed:
size = float(
    trade.get("size")     // CLOB
    or trade.get("amount") // Data API
    or 0
)
usdc = trade.get("usdcSize") \
       or (size * price)

Timestamps

// Three different timestamp fields:
ts = (
    trade.get("timestamp")  // most common
    or trade.get("createdAt") // Data API
    or trade.get("matchTime") // CLOB
)
Price Scale Warning
CLOB returns prices on a 0–1 scale. Gamma outcomePrices uses a 0–100 scale. The Data API's /pricing/* endpoints use 0–1. The accounting snapshot uses 6 decimal places. Always normalize: if price <= 1: price *= 100

What's Free vs. Authenticated
No Auth Required: All Gamma endpoints, CLOB read endpoints (book, prices, midpoint, trades, timeseries), almost all Data API endpoints (positions, trades, leaderboard, pricing, orderbook, profiles, search, comments, accounting).
Auth Required: CLOB order management (place, cancel, active orders), CLOB user WebSocket channel, Bridge deposit/withdrawal operations.

Practical Code Patterns

Real patterns from this project for working with the APIs.

Async Client Pattern

import httpx

class PolymarketClient:
    async def __aenter__(self):
        self.client = httpx.AsyncClient(
            timeout=30.0
        )
        return self

    async def get_markets(self, limit=100):
        r = await self.client.get(
            "https://gamma-api.polymarket.com"
            "/markets",
            params={
                "limit": limit,
                "active": True,
                "order": "volume24hr",
                "ascending": False
            }
        )
        return r.json()

# Usage
async with PolymarketClient() as client:
    markets = await client.get_markets()

Trade Execution (py-clob-client)

from py_clob_client.client import ClobClient

client = ClobClient(
    host="https://clob.polymarket.com",
    key=POLY_PRIVATE_KEY,    // L1

    signature_type=2,        // POLY_PROXY

)

# Generate L2 creds (once)
creds = client.create_or_derive_api_creds()
client.set_api_creds(creds)

# Place a market buy order
order = client.create_and_post_order({
    "tokenID": token_id,
    "side": "BUY",
    "price": 0.55,
    "size": 100
})

Alerting & Notifications

Pushing detected signals to external services.

Email via Postmark

import httpx

async def send_alert(subject, body):
    await httpx.AsyncClient().post(
        "https://api.postmarkapp.com/email",
        headers={
            "X-Postmark-Server-Token":
                POSTMARK_API_TOKEN,
            "Content-Type":
                "application/json"
        },
        json={
            "From":     POSTMARK_FROM_EMAIL,
            "To":       ALERT_EMAIL,
            "Subject":  subject,
            "HtmlBody": body
        }
    )
Required Environment
POSTMARK_API_TOKEN=your-token
POSTMARK_FROM_EMAIL=alerts@you.com
ALERT_EMAIL=you@email.com

Gotchas & Tips

Hard-won lessons from building on these APIs, cross-checked against community experiences.

Price Scale Mismatch
CLOB & Data API pricing return 0–1, Gamma outcomePrices returns 0–100, accounting uses 6 decimal places. Always check and normalize.
Inconsistent Field Names
Never assume a single field name. Always use fallback chains with .get() and or for wallet addresses, market IDs, sizes, and timestamps.
L1 vs L2 Auth Confusion
L1 (EIP-712) is for key management. L2 (HMAC) is for API requests. But even with L2, order payloads still need the private key for signing. You need both.
Missing usdcSize
Not all trade objects include usdcSize. Compute it yourself: usdc = trade.get("usdcSize") or (size * price)
Batch Order Limits
Batch order size was increased from 5 to 15 orders per call in 2025. Use POST /orders for batch placement.
Polygon Chain
Polymarket runs on Polygon (chain ID 137). The py-clob-client needs this explicitly. Three signature types: EOA, POLY_PROXY, GNOSIS_SAFE.
Geographic Restrictions
The CLOB enforces geoblocking. Check geographic restrictions before placing orders. Some regions are blocked from trading.
Data API vs CLOB Overlap
The Data API now has its own /pricing/* and /orderbook/* endpoints. These can be easier to use than CLOB equivalents for read-only operations.
Response Deduplication
When merging data from multiple APIs, dedup on composite keys: {trader}:{market}:{price}:{shares}:{ts}
Strapi Is Legacy
The Strapi endpoint (strapi-matic.poly.market) is legacy. Use Data API /user/activity instead. Only fall back to Strapi when everything else fails.
